How it works. Bridge runs a live tool loop: scan_variances → decompose_line (volume/ rate/ mix, program_mix on margin) → get_driver → cross_reference_accounts, with get_line_history to separate one-offs from trends. The resolvers behind those tools are the only thing it sees, so the model underneath can change without touching the agent.
