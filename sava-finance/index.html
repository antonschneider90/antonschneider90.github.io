About This Model

How this dashboard connects to the Google Sheets model

Data source

This dashboard reads live from a Google Sheets model which is "Published to the web". On page load, the dashboard fetches the key tabs in parallel: Current Year Overview, Forecast Years Overview, RFE_Base, RFE_Bear, RFE_Bull, Assumptions, Headcount Plan, Revenue Build, Summary.

Scenario toggle

The Bear / Base / Bull toggle in the top bar controls every tab. Switching is instant — all three scenario engines (RFE_Bear / RFE_Base / RFE_Bull) are pre-fetched.

Where to change assumptions

All assumptions live in the Assumptions tab of the Sheet. Each scenario has its own column (D = Bear, E = Base, F = Bull). The active scenario for the Sheet's own CYO/FYO views is set in C8 — this dashboard ignores it and uses the toggle instead.

Key cells: C9 last month of actuals · D/E/F 12, 14, 16 launch months · D/E/F 115 steady-state HC adds · D/E/F 116 pre-launch HC adds · D/E/F 102 opening cash · D/E/F 103 runrate cushion months.

When to hit Refresh

Click the Refresh button in the top right whenever you edit the Sheet and want the dashboard to pick up changes. Google Sheets typically takes 1-5 minutes to republish CSV endpoints after a change — wait a minute and try again if numbers don't update.

You do NOT need to refresh when toggling scenarios — that's instant, client-side.

Data integrity check

The Data check badge in the top bar reads from the model's 67-check integrity row. Green OK means all checks pass; red ERROR indicates a mismatch — open the Sheet's Checks tab to investigate.

Model architecture

Top-down: Assumptions → Driver tabs (Revenue Build, COGS & Mfg, Opex Build, Headcount Plan) → RFE_Base / Bear / Bull → CYO & FYO overviews. 67 integrity checks tie outputs back to inputs.
